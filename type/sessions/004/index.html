5-minute catch-up

Missed it? Here's the gist.

Speed and leverage (Alec's angle). Alec used Codex to pull a full prospect list through the SpyFu API in about 10 minutes, work that would have taken a VA a couple days to a week. He paired it with Apify, a library of off-the-shelf scrapers for pulling data off Instagram, YouTube, LinkedIn, and Facebook, and n8n for building workflows you can see and debug step by step. The throughline: anything you would hand to a VA is a candidate for AI, and the real cost saved is lost momentum, not the hourly rate.

Persistent memory and creative output (Justin's angle). Justin showed Hermes, a desktop AI agent that remembers everything about you, switches between models (frontier, cheap like Kimi, free, and local), saves reusable skills so repeat tasks cost fewer tokens, and runs cron jobs like his daily AI-news scout that emails him the top headlines at 3:30 every afternoon. He then walked through Midjourney for creative ideation (merch mockups, recreating images from a prompt) and Claude Design for turning Figma and Canva mockups into production-grade sites, the same way he and Alec built Slow Hand Remodeling and the AI4NTP site.

The tools that quietly compound (the shared finale). Both hosts use Wispr Flow to talk instead of type, with live auto-correction as you speak, and Alec showed the PLAUD Note, a small hardware recorder that captures phone calls and in-person conversations beyond Zoom and pipes them into tools like Google Drive and Claude. The session closed by querying James's agentic GoHighLevel CRM out loud to surface his highest-value prospects to follow up with, a happy medium between going all-in on AI and clinging to an old-school CRM. As Alec put it, this is the worst it will ever be.

FAQ

Questions from the room.

How do I know that what the AI delivered is actually valid?
This came up live (from a live attendee, Rolando). The short answer is that you stay in the loop and verify the output, you do not blindly trust it. This is exactly why Alec walked through n8n: in a visual workflow you can see every step, find where data is leaking or broken, and check the result against reality. Treat AI output like work from a new hire. You review it before you ship it.
When should I use n8n versus Claude Code?
A live attendee (David Toledo) asked whether Alec still recommends n8n over Claude Code because the debugging is more visual. Alec said it is case by case. If you have never built these workflows or you are not technical, n8n is a great starting point because it is visual and teaches you the fundamentals (what a system prompt is, how to make an HTTP request, where things break). Once you are comfortable, use Claude Code for anything complex and reach for n8n when you just need to wire a few tools together quickly.
Wispr Flow or Super Whisper, which one should I use?
Another question from a live attendee (David Toledo), who uses Super Whisper and was hesitant to switch. Alec's honest answer: he has not used Super Whisper, but he assumes it does the same thing (press a hotkey, talk, and it transcribes). His rule of thumb is simple. If Super Whisper is working well for you, there is no reason to switch. If you are running into problems, Wispr Flow has been great for him.
Are these scrapers (like Apify) against terms of service?
A live attendee (Erika) raised this during the Apify demo, and we want to be straight with you: we did not fully address it on the call, so here is a careful answer. Scraping publicly available data sits in a genuinely gray area legally, and every platform has its own terms of service that you are responsible for reading and respecting. Some sites prohibit scraping outright, others allow it within limits, so check each platform's terms before you run anything and do not collect private or personal data you are not entitled to. This is not legal advice. If you are operating at any real scale, talk to a lawyer.
Big picture: Hermes versus Claude Code, when do I use which?
A live attendee (Jayme) asked for the 50,000-foot view, and both hosts answered. Use Hermes (or OpenClaw) for anything scheduled or always-on: cron jobs, recurring marketing tasks, persistent memory that remembers everything about you. Use Claude Code as your daily driver, the place you go to sit down and build something hands-on with your environment and API keys set up. Alec put it simply: if it runs all the time, use Hermes or OpenClaw; if you are going in to get focused work done, use Claude Code.
Do I need to be technical to use a tool like Codex?
No. Alec was clear on this during the Codex demo. You do not need any developer or engineering background, and if something confuses you, you can just ask Codex what to do and it will help you figure it out. The only real rule is to get in there and try it without exposing sensitive data or API keys.
Can I feed in a database of zip codes and get the demographics of the people who live there?
A live attendee (Jean) asked this during the Hermes demo. Justin's answer was honest: yes, there is a really robust tool that people use for exactly this kind of local marketing, but he could not name it on the spot. He asked Jean to send a follow-up email so he could share it directly. We are not going to guess at a name here, so if this is your use case, email us and we will point you to the right tool.
What's the free bonus for showing up live?
Because you came live, you get a free Hermes training and setup, run as weekend office hours, a 30-minute to one-hour boot camp to get you set up and fine-tuned. If you prefer OpenClaw instead, Ian is hosting that setup session, and you are welcome to join both.
