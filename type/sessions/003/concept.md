## The promise

In under an hour, you'll know **step by step how to set up your own OpenClaw**, your own AI employee. Ian walks through his exact setup so the recording becomes the reference clip people point back to ("here's how I did it").

## The sizzle (marketing angle)

- **Your own AI employee. It never sleeps.** Get more done, more efficient, not another Excel spreadsheet.
- **Relational AI:** treat it like a human (it's trained to work with humans), and it acts like an employee, not a robot you micromanage.
- **It's easier to set up than you think** (Alec's point): essentially one command, then step by step, with a few nuances (e.g. spacebar to pick skills).
- **The mind-blow** (Ian): how *proactive* it is. Other tools tell you to "go get this API key." OpenClaw just does it ("I set you up a Supabase, downloaded what I needed"). The lack of obvious limits is what keeps Ian up at night (in a good way).

## What we'll cover

1. **Set it up, step by step.** The one-command start, then the guided setup. Connect it through **Telegram** (Ian's setup); note you can also use WhatsApp, Slack, or Messages.
2. **Hosted vs local.**
   - **VPS (online hosted):** always-on, for building websites, marketing engines, apps/software that live online.
   - **Mac Mini / Mac Studio (local):** for private/sensitive work and local apps. The setup is the same either way.
3. **Security.** The #1 question in the market. What to be careful of, how to mitigate it, and how to avoid exposing sensitive data (we'll be careful not to expose any on the live).
4. **Models (light touch).** Which models you can use, how to switch them, what a local model is and when to use it. Model-matching is part of setup; deeper model talk is a future episode.
5. **Make it your AI employee.** The relational-AI philosophy and real use cases (websites, marketing engines, private local apps).
