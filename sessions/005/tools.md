# Episode 005: tools and resources mentioned

"Three real companies, built with AI in one hour," aired June 30, 2026. Moderator Brett Haralson. Builders Ian Kilpatrick, Justin Novak, and Alec Saluga.

The live chat asked for the tools grouped by function, so this list is organized around how each tool was used rather than only who mentioned it.

## Ideation and reference gathering

**[ChatGPT](/ai-tools/chatgpt)** - Used by Ian for concepting Sunday Crumb and by Alec as part of the early ideation loop before building. *Mentioned by Ian and Alec.*

**[Claude](/ai-tools/claude)** - Used for brand and app ideation, copy, critique, and design translation. *Mentioned by Alec, Justin, and Ian.*

**[Behance](/ai-tools/behance)** - A creative reference source Justin used while looking for Milk and Cookies brand direction. *Mentioned by Justin.*

**Crumbl** - The national cookie brand Alec used as visual inspiration for the Crumb B2B page structure and product photography. *Referenced by Alec and Brett.*

**SusieCakes, Jolene, and Creme** - Bakery brand references that came up during the early scan for examples and inspiration. *Mentioned in the transcript and chat.*

**[FireShot](/ai-tools/fireshot)** - Ian's recommended Chrome extension for capturing a full-page screenshot, which Alec used to pull a complete Crumbl reference page into his prompt context. *Mentioned by Alec and Ian.*

**[DaFont](/ai-tools/dafont)** - A font discovery site Justin mentioned while thinking through the Milk and Cookies brand. *Mentioned by Justin.*

## Brand, visuals, and product mockups

**[Midjourney](/ai-tools/midjourney)** - Justin used Midjourney to generate early logo and brand directions for Milk and Cookies from a spoken prompt. *Demoed by Justin.*

**[Claude Design](/ai-tools/claude)** - Justin used Claude Design to rebuild the Milk and Cookies visual identity, including logo, color palette, typography, brand voice, and an MVP design direction. *Demoed by Justin.*

**[Figma](/ai-tools/figma)** - Justin used Figma as an ideation surface before moving into the AI build. *Mentioned by Justin.*

**[Higgsfield](/ai-tools/higgsfield)** - Alec used Higgsfield for logo generation and AI UGC concepts, plus as a hub for running different image models. *Demoed by Alec.*

**[Nano Banana / Gemini image generation](/ai-tools/gemini)** - Alec used Gemini's image generation capability, referred to as Nano Banana, for the cookie-logo mockup concept inside Crumb B2B. *Demoed by Alec.*

**[Grok](/ai-tools/grok) and [ChatGPT image models](/ai-tools/chatgpt)** - Mentioned by Alec as models available inside Higgsfield for swapping when one model gives weak output. *Mentioned by Alec.*

## App and website building

**[Claude Code](/ai-tools/claude-code)** - Justin used Claude Code inside VS Code to build the Milk and Cookies marketplace flow from a plain-English prompt. *Demoed by Justin.*

**[VS Code](/ai-tools/visual-studio-code)** - Justin's local coding environment for the Milk and Cookies build. *Mentioned by Justin and in chat.*

**[Lovable](/ai-tools/lovable)** - Alec used Lovable to build Crumb B2B because it is an easy on-ramp for non-technical people and can publish without a separate GitHub/Vercel flow. *Demoed by Alec.*

**[OpenClaw](/ai-tools/openclaw)** - Ian ran multiple OpenClaw agents connected to the same server, one working on brand and one on site functionality. *Demoed by Ian.*

**[GitHub](/ai-tools/github)** - Mentioned by Alec while explaining that Lovable can publish without manually pushing to GitHub first. *Mentioned by Alec.*

**[Vercel](/ai-tools/vercel)** - Mentioned as the normal deployment layer for custom apps, and Justin shared the Milk and Cookies Vercel app link in chat. *Mentioned by Alec and Justin.*

**Milk and Cookies prototype** - Justin's live build artifact: https://milk-and-cookies-t-minus-studios.vercel.app. *Shared by Justin in chat.*

## Prompting and workflow acceleration

**[Wispr Flow](/ai-tools/wispr-flow)** - Justin and Alec used voice dictation to talk through prompts faster than typing, especially for product briefs and Lovable prompts. *Demoed by Justin and Alec.*

**Skill files** - Alec showed how a saved Markdown prompt, what he calls a skill file, can make repeat tasks like logo generation more consistent. *Demoed by Alec.*

**Screenshots as context** - Ian emphasized giving AI tools reference screenshots and plain-language design critique so the output moves past the default AI-template look. *Explained by Ian.*

## Production stack mentioned

**[Postgres](/ai-tools/postgres)** - Justin said the production version of Milk and Cookies would likely use Postgres for the database. *Shared by Justin in chat.*

**REST / [tRPC](/ai-tools/trpc)** - Justin mentioned REST or tRPC as the application API layer for a production build. *Shared by Justin in chat.*

**[Stripe](/ai-tools/stripe)** - Justin mentioned Stripe for subscription payments. *Shared by Justin in chat.*

**[Twilio](/ai-tools/twilio)** - Justin mentioned Twilio for SMS and push notifications. *Shared by Justin in chat.*
