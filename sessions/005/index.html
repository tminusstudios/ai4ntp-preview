FAQ

Questions from the room.

What did the audience choose to build?
The room chose a bakery-style business. People suggested several ideas in chat, but baked goods kept coming up, so Brett locked the prompt and let each builder choose a market: Alec in LA, Justin in New York, and Ian in Nashville.
Who won the build-off?
The final poll was framed as a vanity vote, not a formal judging panel. From the transcript, Alec's Crumb B2B appeared to win the room, with Ian's Nashville truck close behind and Justin's Milk and Cookies still getting called out as a strong idea.
What did Justin build?
Justin built Milk and Cookies, a marketplace that connects local farmers, ghost-kitchen bakers, delivery drivers, and customers. The final demo showed a customer building a weekly box, bakers uploading inventory, and a driver accepting orders.
What did Alec build?
Alec built Crumb B2B, a B2B sales and retention product for sending logo-printed cookies to prospects or clients. The flow let a user choose a cookie drop, upload a logo, generate a mockup, and place an order.
What did Ian build?
Ian built Sunday Crumb, a Nashville bakery truck concept where customers scan a QR code, invent a pastry, choose fillings and toppings, name the creation, and submit it to the truck. He also built an admin view and brand system around the idea.
Do I need to know how to code to do what they did?
No. Justin said he did not write a single line of code by hand, he talked to his computer with Wispr Flow and let Claude Code do the implementation. Alec's answer at the end was the most honest version: either no code was written by the human, or AI wrote thousands of lines for them.
Which tool should a beginner start with?
For website and app prototypes, Alec recommended Lovable as the easiest on-ramp because you can paste in a prompt and reference screenshot, then publish without handling GitHub or Vercel yourself. Justin's path is more powerful but more local: Claude Code inside VS Code. Ian's OpenClaw path is better when you want multiple agents working on a server.
What backend would a real production version need?
For Justin's Milk and Cookies concept, he answered in chat that the production stack would likely include Postgres, REST or tRPC, Stripe for subscriptions, and Twilio for SMS and push notifications. In the session, the builds were prototypes and mockups, not fully production-hardened apps.
